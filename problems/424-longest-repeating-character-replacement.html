    Sliding Window / Hash Table

    424. Longest Repeating Character Replacement

    最多替换 k 个字符后，求能得到的最长重复字符子串长度。

    Medium滑动窗口哈希表字符串

    题目白话解释

    在一个窗口里，如果把除了出现次数最多的字符以外的字符都替换掉，窗口就能变成同一个字符。替换次数不能超过 k。

    核心思路一句话

    维护窗口内最高频字符次数，当 窗口长度 - 最高频次数 > k 时收缩窗口。

    为什么会想到这个方法

    想把窗口变成全相同字符，最省替换的方法一定是保留出现最多的字符，替换其他字符。因此只需要知道窗口长度和最高频次数。

    Python 前置知识

    • 字典可记录每个字符出现次数。
    • max_count 保存窗口历史最高频次数。
    • 窗口长度是 right - left + 1。

    解题步骤

    1. 右指针加入字符并更新计数。
    2. 更新最高频字符次数。
    3. 如果需要替换的字符数超过 k，左指针右移。
    4. 每一步更新最长窗口长度。

    完整 Python 代码

    leetcode.py
    class Solution:
    def characterReplacement(self, s: str, k: int) -> int:
        count = {}
        left = 0
        max_count = 0
        best = 0

        for right, ch in enumerate(s):
            count[ch] = count.get(ch, 0) + 1
            max_count = max(max_count, count[ch])

            while right - left + 1 - max_count > k:
                count[s[left]] -= 1
                left += 1

            best = max(best, right - left + 1)

        return best

    逐行解释

    • count[ch]：统计窗口内字符数量。
    • max_count：窗口中某个字符的最高出现次数。
    • right - left + 1 - max_count：需要替换的字符数量。
    • 超过 k 就移动左指针。
    • best 保存合法窗口最大长度。

    示例执行过程

    s = "AABABBA", k = 1。窗口 "AABA" 中 A 出现 3 次，只需替换一个 B，长度 4 合法。

    时间复杂度

    O(n)，左右指针线性移动。

    空间复杂度

    O(1)，如果只包含大写英文字母，字典最多 26 个键。

    易错点

    max_count 不需要在左指针移动时强制下降。它可能偏大，但不会影响最终最大长度，只会让窗口稍晚收缩。

    ACM 模式输入输出改写

    输入第一行字符串，第二行整数 k，输出最长长度。

    acm.py
    import sys

def character_replacement(s, k):
    count = {}
    left = 0
    max_count = 0
    best = 0
    for right, ch in enumerate(s):
        count[ch] = count.get(ch, 0) + 1
        max_count = max(max_count, count[ch])
        while right - left + 1 - max_count > k:
            count[s[left]] -= 1
            left += 1
        best = max(best, right - left + 1)
    return best

lines = sys.stdin.read().strip().split()
s = lines[0]
k = int(lines[1])
print(character_replacement(s, k))
