    41. First Missing Positive

    找出数组中缺失的最小正整数。

    Hard哈希表数组原地哈希

    题目白话解释

    给你一个乱序数组，问从 1 开始数，哪个正整数最先没有出现。

    核心思路一句话

    把值为 x 的数字尽量放到下标 x - 1 的位置，再从前往后找第一个不匹配的位置。

    为什么会想到这个方法

    长度为 n 的数组，答案只可能在 1..n+1。如果 1 到 n 都在正确位置，那答案就是 n+1；否则第一个位置不对的下标就能推出缺失值。

    Python 前置知识

    • 列表元素可以交换：nums[i], nums[j] = nums[j], nums[i]。
    • while 用于把当前位置数字连续放回正确位置。
    • 要判断目标位置是否已经有相同值，避免死循环。

    解题步骤

    1. 遍历每个位置。
    2. 如果当前值在 1..n 内，就尝试放到它应该在的位置。
    3. 重复交换，直到当前位置无法再放。
    4. 再次遍历数组。
    5. 第一个 nums[i] != i + 1 的位置，答案就是 i + 1。

    完整 Python 代码

    leetcode.py
    class Solution:
    def firstMissingPositive(self, nums: list[int]) -> int:
        n = len(nums)

        for i in range(n):
            while 1 <= nums[i] <= n and nums[nums[i] - 1] != nums[i]:
                target = nums[i] - 1
                nums[i], nums[target] = nums[target], nums[i]

        for i in range(n):
            if nums[i] != i + 1:
                return i + 1

        return n + 1

    逐行解释

    • 1 <= nums[i] <= n：只处理可能影响答案的正数。
    • target = nums[i] - 1：数字 x 应该放到 x-1。
    • 交换后继续检查当前位置的新数字。
    • 第二轮遍历寻找第一个不匹配位置。
    • 全部匹配说明缺失的是 n + 1。

    示例执行过程

    [3,4,-1,1] 经过交换后，1 放到下标 0，3 放到下标 2。下标 1 没有放 2，所以答案是 2。

    时间复杂度

    O(n)，每个数字最多被放回正确位置一次。

    空间复杂度

    O(1)，原地交换。

    易错点

    交换条件里必须判断 nums[nums[i] - 1] != nums[i]，否则遇到重复数字会死循环。

    ACM 模式输入输出改写

    输入第一项为 n，后面跟着数组，输出缺失的最小正整数。

    acm.py
    import sys

def first_missing_positive(nums):
    n = len(nums)
    for i in range(n):
        while 1 <= nums[i] <= n and nums[nums[i] - 1] != nums[i]:
            target = nums[i] - 1
            nums[i], nums[target] = nums[target], nums[i]
    for i in range(n):
        if nums[i] != i + 1:
            return i + 1
    return n + 1

data = sys.stdin.read().strip().split()
n = int(data[0])
nums = list(map(int, data[1:1 + n]))
print(first_missing_positive(nums))
