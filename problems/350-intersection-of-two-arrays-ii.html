    350. Intersection of Two Arrays II

    求两个数组的交集，重复元素按出现次数保留。

    Easy哈希表双指针数组

    题目白话解释

    两个数组里都出现的数字要输出。如果某个数字在两个数组中分别出现 2 次和 3 次，交集里只能放 2 次。

    核心思路一句话

    先统计较短数组的数字次数，再用另一个数组逐个抵消。

    为什么会想到这个方法

    因为重复次数不能丢，普通集合会把重复信息去掉。用哈希表记录剩余次数，刚好能处理“最多取共同出现次数”的要求。

    Python 前置知识

    • dict.get(key, 0) 可以安全读取默认值。
    • 列表 append 用于收集答案。
    • 可以交换两个数组，让计数字典尽量小。

    解题步骤

    1. 让 nums1 指向较短数组。
    2. 统计 nums1 中每个数字出现次数。
    3. 遍历 nums2。
    4. 如果当前数字还有剩余次数，就加入答案并减 1。
    5. 返回答案列表。

    完整 Python 代码

    leetcode.py
    class Solution:
    def intersect(self, nums1: list[int], nums2: list[int]) -> list[int]:
        if len(nums1) > len(nums2):
            nums1, nums2 = nums2, nums1

        count = {}
        for num in nums1:
            count[num] = count.get(num, 0) + 1

        ans = []
        for num in nums2:
            if count.get(num, 0) > 0:
                ans.append(num)
                count[num] -= 1

        return ans

    逐行解释

    • if len(nums1) > len(nums2)：优先统计较短数组，节省空间。
    • count[num]：记录每个数字还能使用几次。
    • ans = []：保存交集结果。
    • count.get(num, 0) > 0：只有双方都有剩余次数才加入。
    • count[num] -= 1：避免同一次数被重复使用。

    示例执行过程

    nums1 = [1,2,2,1]，nums2 = [2,2]。统计后 2 有 2 次，遍历第二个数组时两个 2 都能加入，答案是 [2,2]。

    时间复杂度

    O(m + n)，两个数组各遍历一次。

    空间复杂度

    O(min(m, n))，只统计较短数组。

    易错点

    不能直接用 set 求交集，因为这题要求保留重复次数。

    ACM 模式输入输出改写

    输入 m n，后面分别是两个数组，输出交集。

    acm.py
    import sys

def intersect(nums1, nums2):
    if len(nums1) > len(nums2):
        nums1, nums2 = nums2, nums1
    count = {}
    for num in nums1:
        count[num] = count.get(num, 0) + 1
    ans = []
    for num in nums2:
        if count.get(num, 0) > 0:
            ans.append(num)
            count[num] -= 1
    return ans

data = sys.stdin.read().strip().split()
m, n = map(int, data[:2])
nums1 = list(map(int, data[2:2 + m]))
nums2 = list(map(int, data[2 + m:2 + m + n]))
print(*intersect(nums1, nums2))
