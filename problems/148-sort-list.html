    Linked List

    148. Sort List

    在 O(n log n) 时间内排序链表。

    Medium链表归并排序

    题目白话解释

    链表不能像数组一样高效随机访问，所以更适合归并排序，而不是快速排序或堆排序。

    核心思路一句话

    快慢指针切成两半，递归排序两半，再合并两个有序链表。

    为什么会想到这个方法

    归并排序只需要顺序扫描和改指针，天然适合链表；切分用快慢指针可以找到中点。

    Python 前置知识

    • 快慢指针找中点。
    • 递归排序左右两段。
    • 合并阶段和 21 题相同。

    解题步骤

    1. 如果链表为空或只有一个节点，直接返回。
    2. 用快慢指针找到中点前一个节点。
    3. 断开链表，得到左右两段。
    4. 分别递归排序左右段。
    5. 合并两个有序链表。

    完整 Python 代码

    leetcode.py
    class Solution:
    def sortList(self, head):
        if not head or not head.next:
            return head

        slow, fast = head, head.next
        while fast and fast.next:
            slow = slow.next
            fast = fast.next.next

        mid = slow.next
        slow.next = None

        left = self.sortList(head)
        right = self.sortList(mid)
        return self.merge(left, right)

    def merge(self, a, b):
        dummy = ListNode(0)
        tail = dummy
        while a and b:
            if a.val <= b.val:
                tail.next = a
                a = a.next
            else:
                tail.next = b
                b = b.next
            tail = tail.next
        tail.next = a if a else b
        return dummy.next

    逐行解释

    • 递归出口是空链表或单节点。
    • slow.next = None 把链表切成两段。
    • merge 复用合并两个有序链表模板。
    • 每层递归做线性合并。

    示例执行过程

    4 -&gt; 2 -&gt; 1 -&gt; 3 切成两半排序，再合并成 1 -&gt; 2 -&gt; 3 -&gt; 4。

    时间复杂度

    O(n log n)。

    空间复杂度

    O(log n)，递归栈空间。

    易错点

    • 必须断开左右两段，否则递归不会收敛。
    • 找中点时 fast 从 head.next 开始更容易切成均衡两段。
    • 合并时不要新建过多节点，直接改指针即可。

    ACM 模式输入输出改写

    数组输入时可以直接排序输出；链表版面试要掌握归并思路。

    acm.py
    # input() 读取一行，split() 切分，map() 转整数，print() 输出结果
nums = list(map(int, input().split()))
nums.sort()
print(*nums)
