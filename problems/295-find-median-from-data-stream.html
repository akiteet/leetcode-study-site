    295. Find Median from Data Stream

    不断加入数字，并随时返回当前中位数。

    Hard堆数据流

    题目白话解释

    数字一个个到来，不能每次都重新排序；要维护一个结构，让中间位置的数容易拿到。

    核心思路一句话

    用两个堆：大根堆保存较小一半，小根堆保存较大一半。

    为什么会想到这个方法

    中位数只和中间两个数有关。把数据分成左右两半，并保持两边数量平衡，就能快速得到中位数。

    Python 前置知识

    • Python 没有大根堆，可把数字取负后放进 heapq。
    • 两个堆的大小差最多为 1。
    • 左堆所有值都应小于等于右堆所有值。

    解题步骤

    1. 新数字先放入左堆。
    2. 把左堆最大值移动到右堆，保证左右顺序。
    3. 如果右堆数量更多，再移一个回左堆。
    4. 查询时，如果左堆更多，返回左堆堆顶。
    5. 如果两边一样多，返回两个堆顶平均值。

    完整 Python 代码

    leetcode.py
    import heapq

class MedianFinder:
    def __init__(self):
        self.small = []
        self.large = []

    def addNum(self, num: int) -> None:
        heapq.heappush(self.small, -num)
        heapq.heappush(self.large, -heapq.heappop(self.small))
        if len(self.large) > len(self.small):
            heapq.heappush(self.small, -heapq.heappop(self.large))

    def findMedian(self) -> float:
        if len(self.small) > len(self.large):
            return -self.small[0]
        return (-self.small[0] + self.large[0]) / 2

    逐行解释

    • small 用负数模拟大根堆，保存较小一半。
    • large 是小根堆，保存较大一半。
    • 每次加入后都重新平衡两个堆。
    • 中位数来自一个堆顶或两个堆顶平均值。

    示例执行过程

    依次加入 1、2 时中位数是 1.5；再加入 3 后中位数是 2。

    时间复杂度

    添加数字 O(log n)，查询中位数 O(1)。

    空间复杂度

    O(n)。

    易错点

    • 大根堆要用负数模拟。
    • 两个堆大小差不能超过 1。
    • 偶数个数字时要返回平均值。

    ACM 模式输入输出改写

    输入一串数字，每加入一个数就输出当前中位数。

    acm.py
    # input() + split() + map() 读取数据流，print() 输出每一步中位数
import heapq

nums = list(map(int, input().split()))
small, large = [], []

for num in nums:
    heapq.heappush(small, -num)
    heapq.heappush(large, -heapq.heappop(small))
    if len(large) > len(small):
        heapq.heappush(small, -heapq.heappop(large))

    if len(small) > len(large):
        print(float(-small[0]))
    else:
        print((-small[0] + large[0]) / 2)
