    55. Jump Game

    判断能否从数组起点跳到最后一个位置。

    Medium贪心数组

    题目白话解释

    每个位置的数字表示最多能跳多远，你可以少跳，但不能超过这个距离。

    核心思路一句话

    维护当前能到达的最远位置，只要遍历过程中当前位置可达，就尝试更新最远距离。

    为什么会想到这个方法

    题目只问能不能到，不问怎么跳。只要知道当前最远能覆盖到哪里，就能判断后续位置是否有机会继续扩展。

    Python 前置知识

    • 贪心常维护一个当前最优边界。
    • max_reach 表示目前最远可达下标。
    • 如果当前下标大于 max_reach，说明断开了。

    解题步骤

    1. 初始化 max_reach = 0。
    2. 从左到右遍历每个下标。
    3. 如果 i > max_reach，返回 False。
    4. 更新 max_reach = max(max_reach, i + nums[i])。
    5. 如果最远距离覆盖末尾，返回 True。

    完整 Python 代码

    leetcode.py
    class Solution:
    def canJump(self, nums: list[int]) -> bool:
        max_reach = 0
        for i, jump in enumerate(nums):
            if i > max_reach:
                return False
            max_reach = max(max_reach, i + jump)
            if max_reach >= len(nums) - 1:
                return True
        return True

    逐行解释

    • i > max_reach 表示当前位置根本走不到。
    • i + jump 是从当前位置能扩展到的最远点。
    • 只要覆盖最后一个下标，就可以提前返回。
    • 单元素数组天然可达。

    示例执行过程

    [2,3,1,1,4] 中，从 0 可跳到 1，再从 1 覆盖到最后。

    时间复杂度

    O(n)。

    空间复杂度

    O(1)。

    易错点

    • 每个位置是最多跳，不是必须跳。
    • 不要用回溯暴力枚举路径。
    • 遇到 0 不一定失败，要看它之前是否能被跨过去。

    ACM 模式输入输出改写

    输入数组，输出是否可以到达最后一个位置。

    acm.py
    # input() + split() + map() 读取数组，print() 输出布尔值
nums = list(map(int, input().split()))

max_reach = 0
ok = True
for i, jump in enumerate(nums):
    if i > max_reach:
        ok = False
        break
    max_reach = max(max_reach, i + jump)

print(ok)
