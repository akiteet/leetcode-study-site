    128. Longest Consecutive Sequence

    找出数组中最长的连续整数序列长度，数字不要求在原数组里相邻。

    Medium哈希表数组

    题目白话解释

    给你一堆乱序数字，问能拼出的最长连续段有多长。例如 [100,4,200,1,3,2] 里有 1,2,3,4，答案是 4。

    核心思路一句话

    把数字放进集合，只从“连续段起点”开始向后数。

    为什么会想到这个方法

    判断一个数是否存在是核心操作，集合查找平均是 O(1)。如果 num - 1 不存在，num 才可能是一段连续序列的起点，避免重复从中间数字开始数。

    Python 前置知识

    • set(nums) 可以去重并支持快速查找。
    • x in set 平均时间复杂度是 O(1)。
    • while 适合不断向后扩展连续数字。

    解题步骤

    1. 把所有数字放入集合。
    2. 遍历集合中的每个数字。
    3. 如果 num - 1 不在集合中，说明它是起点。
    4. 从起点开始不断检查 current + 1 是否存在。
    5. 更新最长长度。

    完整 Python 代码

    leetcode.py
    class Solution:
    def longestConsecutive(self, nums: list[int]) -> int:
        num_set = set(nums)
        best = 0

        for num in num_set:
            if num - 1 not in num_set:
                current = num
                length = 1

                while current + 1 in num_set:
                    current += 1
                    length += 1

                best = max(best, length)

        return best

    逐行解释

    • num_set = set(nums)：去重并让查找更快。
    • best = 0：记录目前最长连续长度。
    • if num - 1 not in num_set：只从连续段起点开始数。
    • while current + 1 in num_set：持续向右扩展。
    • best = max(best, length)：保存最大答案。

    示例执行过程

    输入 [100,4,200,1,3,2]。集合包含所有数字。遍历到 1 时发现 0 不存在，于是从 1 数到 4，长度为 4；100 和 200 都只能形成长度 1 的序列。

    时间复杂度

    O(n)。每个数字最多作为起点或扩展数字被处理常数次。

    空间复杂度

    O(n)，集合需要保存所有不同数字。

    易错点

    不要从每个数字都开始向后数，否则会退化成重复扫描。必须先判断它是不是起点。

    ACM 模式输入输出改写

    输入第一项为 n，后面跟着 n 个整数，输出最长连续序列长度。

    acm.py
    import sys

def longest_consecutive(nums):
    num_set = set(nums)
    best = 0
    for num in num_set:
        if num - 1 not in num_set:
            current = num
            length = 1
            while current + 1 in num_set:
                current += 1
                length += 1
            best = max(best, length)
    return best

data = sys.stdin.read().strip().split()
n = int(data[0])
nums = list(map(int, data[1:1 + n]))
print(longest_consecutive(nums))
