    Linked List

    160. Intersection of Two Linked Lists

    判断两个链表是否在某个节点开始共用同一段尾巴。

    Easy链表双指针

    题目白话解释

    相交不是值相等，而是两个链表走到了同一个节点对象。返回这个公共节点；不相交就返回 None。

    核心思路一句话

    两个指针分别走 A+B 和 B+A，长度差会被自动抵消。

    为什么会想到这个方法

    如果两条链表长度不同，直接同步走会错位。让 A 走完后接 B，B 走完后接 A，两者总路程相同，若有交点会同时到达。

    Python 前置知识

    • 链表节点比较要比较对象本身，不是比较 val。
    • None 也可以作为循环结束和换路标记。
    • 双指针可以用来消除长度差。

    解题步骤

    1. 令 p = headA，q = headB。
    2. 两个指针每次向后走一步。
    3. p 走到空后切到 headB，q 走到空后切到 headA。
    4. 当 p == q 时返回该节点，可能是交点，也可能都是 None。

    完整 Python 代码

    leetcode.py
    class Solution:
    def getIntersectionNode(self, headA, headB):
        p, q = headA, headB
        while p != q:
            p = p.next if p else headB
            q = q.next if q else headA
        return p

    逐行解释

    • p 先走链表 A，再走链表 B。
    • q 先走链表 B，再走链表 A。
    • 如果有公共尾部，两个指针会在交点相遇。
    • 如果没有交点，最后会同时变成 None。

    示例执行过程

    A 比 B 长 2 个节点时，切换链表后短的那条会补齐这 2 步，最终对齐公共尾部。

    时间复杂度

    O(m+n)。

    空间复杂度

    O(1)。

    易错点

    • 不要用节点值判断是否相交。
    • 循环条件是 p != q，不是 p.next != q.next。
    • 不相交时也能结束，因为最终都会到 None。

    ACM 模式输入输出改写

    ACM 中没有真实节点引用时，可以把节点编号当作身份，找两个编号序列第一个共同节点。

    acm.py
    # input() 读取一行，split() 切分，map() 转整数，print() 输出结果
a = list(map(int, input().split()))
b = list(map(int, input().split()))

seen = set(a)
ans = -1
for node_id in b:
    if node_id in seen:
        ans = node_id
        break

print(ans)
