    25. Reverse Nodes in k-Group

    每 k 个节点为一组翻转，不足 k 个的尾部保持原样。

    Hard链表分组反转

    题目白话解释

    1 -&gt; 2 -&gt; 3 -&gt; 4 -&gt; 5，当 k = 2 时变成 2 -&gt; 1 -&gt; 4 -&gt; 3 -&gt; 5。

    核心思路一句话

    先确认这一组够 k 个节点，再把这段链表局部反转并接回原链表。

    为什么会想到这个方法

    不足 k 个不能翻转，所以每轮必须先找组尾。找到一组后，反转逻辑可以复用 206 题。

    Python 前置知识

    • 虚拟头节点方便处理第一组。
    • group_prev 表示当前组前一个节点。
    • 局部反转后要重新接上前后两端。

    解题步骤

    1. 用 dummy 指向头节点。
    2. 每轮从 group_prev 往后找第 k 个节点。
    3. 如果找不到，返回结果。
    4. 保存下一组开头 group_next。
    5. 把当前组反转到 group_next 前。
    6. 接回链表并移动 group_prev。

    完整 Python 代码

    leetcode.py
    class Solution:
    def reverseKGroup(self, head, k):
        dummy = ListNode(0, head)
        group_prev = dummy

        while True:
            kth = group_prev
            for _ in range(k):
                kth = kth.next
                if not kth:
                    return dummy.next

            group_next = kth.next
            prev, cur = group_next, group_prev.next
            while cur != group_next:
                nxt = cur.next
                cur.next = prev
                prev = cur
                cur = nxt

            old_head = group_prev.next
            group_prev.next = kth
            group_prev = old_head

    逐行解释

    • kth 用来确认当前组够不够 k 个。
    • group_next 是下一组开头，也是反转后的尾部连接点。
    • 局部反转时 prev 初始为 group_next。
    • old_head 反转后变成这一组尾部。

    示例执行过程

    [1,2,3,4,5]，k=3 时先翻转 [1,2,3]，尾部 [4,5] 不足 3 个保持不变。

    时间复杂度

    O(n)。

    空间复杂度

    O(1)。

    易错点

    • 不足 k 个不能翻转。
    • 反转后组头组尾会互换。
    • 局部反转边界是 group_next，不是 None。

    ACM 模式输入输出改写

    数组输入时可以每 k 个元素一组反转，不足 k 个保留。

    acm.py
    # input() 读取一行，split() 切分，map() 转整数，print() 输出结果
nums = list(map(int, input().split()))
k = int(input())

for start in range(0, len(nums), k):
    if start + k <= len(nums):
        nums[start:start + k] = reversed(nums[start:start + k])

print(*nums)
