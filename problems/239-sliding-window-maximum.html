    Sliding Window / Monotonic Queue

    239. Sliding Window Maximum

    返回每个长度为 k 的滑动窗口中的最大值。

    Hard滑动窗口队列单调队列

    题目白话解释

    一个长度为 k 的窗口从左滑到右，每滑到一个位置，就要快速知道窗口里的最大值。

    核心思路一句话

    用单调递减队列保存可能成为最大值的下标，队首就是当前窗口最大值。

    为什么会想到这个方法

    窗口每次移动只进出一个元素。如果新元素比队尾元素大，队尾元素以后不可能成为最大值，可以直接删掉。这样队列始终保持从大到小。

    Python 前置知识

    • deque 支持两端快速弹出。
    • 队列中保存下标，方便判断是否过期。
    • nums[queue[0]] 是当前最大值。

    解题步骤

    1. 遍历每个下标。
    2. 先移除已经离开窗口的队首下标。
    3. 再从队尾移除所有小于等于当前值的下标。
    4. 把当前下标入队。
    5. 窗口形成后记录队首对应值。

    完整 Python 代码

    leetcode.py
    from collections import deque

class Solution:
    def maxSlidingWindow(self, nums: list[int], k: int) -> list[int]:
        queue = deque()
        ans = []

        for i, num in enumerate(nums):
            while queue and queue[0] <= i - k:
                queue.popleft()

            while queue and nums[queue[-1]] <= num:
                queue.pop()

            queue.append(i)

            if i >= k - 1:
                ans.append(nums[queue[0]])

        return ans

    逐行解释

    • queue[0] <= i - k：队首已经不在窗口内。
    • nums[queue[-1]] <= num：队尾更小，未来不会再当最大值。
    • queue.append(i)：当前下标进入候选队列。
    • i >= k - 1：窗口长度达到 k 后开始记录答案。
    • queue[0] 始终是最大值下标。

    示例执行过程

    nums = [1,3,-1,-3,5,3,6,7], k = 3。第一个窗口 [1,3,-1] 最大值是 3，后续窗口依次得到 [3,3,5,5,6,7]。

    时间复杂度

    O(n)，每个下标最多入队一次、出队一次。

    空间复杂度

    O(k)，队列最多保存一个窗口内的下标。

    易错点

    队列里要保存下标而不是值；只有下标才能判断元素是否已经滑出窗口。

    ACM 模式输入输出改写

    输入第一行 n k，第二行数组，输出每个窗口最大值。

    acm.py
    import sys
from collections import deque

def max_sliding_window(nums, k):
    queue = deque()
    ans = []
    for i, num in enumerate(nums):
        while queue and queue[0] <= i - k:
            queue.popleft()
        while queue and nums[queue[-1]] <= num:
            queue.pop()
        queue.append(i)
        if i >= k - 1:
            ans.append(nums[queue[0]])
    return ans

data = sys.stdin.read().strip().split()
n, k = map(int, data[:2])
nums = list(map(int, data[2:2 + n]))
print(*max_sliding_window(nums, k))
