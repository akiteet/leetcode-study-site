    138. Copy List with Random Pointer

    复制一个带 next 和 random 指针的链表。

    Medium链表哈希表

    题目白话解释

    每个节点除了指向下一个节点，还可能随机指向链表中的任意节点或空。要求深拷贝，不能复用原节点。

    核心思路一句话

    用哈希表建立“原节点 -> 新节点”的映射，再连接新节点的 next 和 random。

    为什么会想到这个方法

    random 可能指向后面的节点，也可能指回前面。先创建所有新节点，就能在第二遍安全地连接任意指针。

    Python 前置知识

    • 字典可以用对象作为 key。
    • 深拷贝表示新链表节点全部是新对象。
    • 处理空指针时要返回 None。

    解题步骤

    1. 如果头节点为空，返回 None。
    2. 第一遍遍历，为每个原节点创建新节点并放入字典。
    3. 第二遍遍历，设置新节点的 next 和 random。
    4. 返回 old_to_new[head]。

    完整 Python 代码

    leetcode.py
    class Solution:
    def copyRandomList(self, head):
        if not head:
            return None

        old_to_new = {}
        cur = head
        while cur:
            old_to_new[cur] = Node(cur.val)
            cur = cur.next

        cur = head
        while cur:
            old_to_new[cur].next = old_to_new.get(cur.next)
            old_to_new[cur].random = old_to_new.get(cur.random)
            cur = cur.next

        return old_to_new[head]

    逐行解释

    • 第一遍只复制节点值。
    • old_to_new[cur] 是当前原节点对应的新节点。
    • dict.get(None) 会返回 None，正好处理空指针。
    • 第二遍再补齐指针关系。

    示例执行过程

    节点 0 的 random 指向节点 2，则新节点 0 的 random 要指向新节点 2，而不是原节点 2。

    时间复杂度

    O(n)。

    空间复杂度

    O(n)，保存映射。

    易错点

    • 不要让新链表的 random 指回原链表节点。
    • 空链表要直接返回 None。
    • random 可能为空，不能直接索引字典。

    ACM 模式输入输出改写

    ACM 可用下标表示 random 指向，输出复制后的值和 random 下标用于检查。

    acm.py
    # input() 读取一行，split() 切分，map() 转整数，print() 输出结果
n = int(input())
nodes = []
for _ in range(n):
    val, random_index = map(int, input().split())
    nodes.append([val, random_index])

copied = [[val, random_index] for val, random_index in nodes]
for val, random_index in copied:
    print(val, random_index)
