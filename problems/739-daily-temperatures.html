    Monotonic Stack

    739. Daily Temperatures

    对每一天，求还要等几天才会遇到更高温度。

    Medium栈单调栈数组

    题目白话解释

    每天都在等未来某天升温。输出数组中每个位置表示从这天开始要等几天；如果之后不会更热，就是 0。

    核心思路一句话

    用单调递减栈保存还没找到更高温度的日期下标。

    为什么会想到这个方法

    每个新温度都可能解决前面一些较低温度的等待问题。栈顶代表最近还没解决的日期，如果当前温度更高，就可以不断弹栈并填答案。

    Python 前置知识

    • 列表可用作栈：append 入栈，pop 出栈。
    • 栈中保存下标，方便计算天数差。
    • 答案数组可先初始化为全 0。

    解题步骤

    1. 创建答案数组和空栈。
    2. 遍历每一天的温度。
    3. 当前温度大于栈顶日期温度时，说明栈顶找到答案。
    4. 弹出栈顶并计算等待天数。
    5. 把当前下标入栈。

    完整 Python 代码

    leetcode.py
    class Solution:
    def dailyTemperatures(self, temperatures: list[int]) -> list[int]:
        ans = [0] * len(temperatures)
        stack = []

        for i, temp in enumerate(temperatures):
            while stack and temp > temperatures[stack[-1]]:
                prev = stack.pop()
                ans[prev] = i - prev
            stack.append(i)

        return ans

    逐行解释

    • ans = [0] * len(...)：默认没有更高温就是 0。
    • stack：保存等待更高温的日期下标。
    • temp > temperatures[stack[-1]]：当前温度解决了栈顶日期。
    • i - prev：等待天数。
    • stack.append(i)：当前日期也开始等待未来更高温。

    示例执行过程

    输入 [73,74,75,71,69,72,76,73]。74 解决 73，75 解决 74，72 解决 69 和 71，76 解决 72 和 75。

    时间复杂度

    O(n)，每个下标最多入栈一次、出栈一次。

    空间复杂度

    O(n)，最坏情况下温度一直下降，所有下标都留在栈中。

    易错点

    栈里放下标，不要只放温度；否则无法计算等待了几天。

    ACM 模式输入输出改写

    输入第一项为 n，后面跟着 n 个温度，输出等待天数数组。

    acm.py
    import sys

def daily_temperatures(temperatures):
    ans = [0] * len(temperatures)
    stack = []
    for i, temp in enumerate(temperatures):
        while stack and temp > temperatures[stack[-1]]:
            prev = stack.pop()
            ans[prev] = i - prev
        stack.append(i)
    return ans

data = sys.stdin.read().strip().split()
n = int(data[0])
temperatures = list(map(int, data[1:1 + n]))
print(*daily_temperatures(temperatures))
