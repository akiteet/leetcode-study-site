    BFS / Queue

    994. Rotting Oranges

    每分钟腐烂橘子会感染四周新鲜橘子，求全部腐烂所需时间。

    MediumBFS队列矩阵

    题目白话解释

    所有腐烂橘子同时向外扩散。每过一分钟，它们会让相邻的新鲜橘子腐烂。问最后需要几分钟，或者是否永远无法全部腐烂。

    核心思路一句话

    从所有初始腐烂橘子同时入队，做多源 BFS。

    为什么会想到这个方法

    “每分钟扩散一层”就是 BFS 的层序遍历。多个腐烂橘子同时扩散，所以一开始要把所有腐烂橘子都放进队列。

    Python 前置知识

    • deque 适合队列。
    • BFS 一层代表一分钟。
    • 可以用 fresh 统计剩余新鲜橘子数量。

    解题步骤

    1. 扫描网格，统计新鲜橘子并收集腐烂橘子。
    2. 队列不空且还有新鲜橘子时，按层扩散。
    3. 腐烂相邻的新鲜橘子并入队。
    4. 每处理一层分钟数加 1。
    5. 最后如果还有新鲜橘子返回 -1。

    完整 Python 代码

    leetcode.py
    from collections import deque

class Solution:
    def orangesRotting(self, grid: list[list[int]]) -> int:
        rows, cols = len(grid), len(grid[0])
        queue = deque()
        fresh = 0

        for r in range(rows):
            for c in range(cols):
                if grid[r][c] == 2:
                    queue.append((r, c))
                elif grid[r][c] == 1:
                    fresh += 1

        minutes = 0
        directions = [(1, 0), (-1, 0), (0, 1), (0, -1)]

        while queue and fresh > 0:
            for _ in range(len(queue)):
                r, c = queue.popleft()
                for dr, dc in directions:
                    nr, nc = r + dr, c + dc
                    if 0 <= nr < rows and 0 <= nc < cols and grid[nr][nc] == 1:
                        grid[nr][nc] = 2
                        fresh -= 1
                        queue.append((nr, nc))
            minutes += 1

        return minutes if fresh == 0 else -1

    逐行解释

    • queue：保存当前会继续扩散的腐烂橘子。
    • fresh：剩余新鲜橘子数量。
    • for _ in range(len(queue))：固定当前分钟要处理的橘子。
    • grid[nr][nc] = 2：新鲜橘子被感染。
    • 还有新鲜橘子则返回 -1。

    示例执行过程

    初始腐烂橘子为第 0 层，第一分钟感染相邻一圈，第二分钟继续向外感染。层数就是分钟数。

    时间复杂度

    O(m * n)，每个格子最多入队一次。

    空间复杂度

    O(m * n)，队列最坏可能保存大量格子。

    易错点

    必须多源 BFS，把所有初始腐烂橘子一起入队。一个一个单独 BFS 会算错时间。

    ACM 模式输入输出改写

    输入第一行 m n，后面 m 行矩阵，元素用空格分隔。

    minutes = 0
