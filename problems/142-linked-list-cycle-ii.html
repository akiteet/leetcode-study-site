    142. Linked List Cycle II

    如果链表有环，返回环的入口节点。

    Medium链表快慢指针

    题目白话解释

    141 题只问有没有环，本题还要找到从哪里开始进入环。

    核心思路一句话

    快慢指针先在环内相遇，再让一个指针回到头部，两者同步走，下一次相遇就是入口。

    为什么会想到这个方法

    快慢指针相遇后，根据路程关系，头节点到入口的距离等于相遇点继续走到入口的距离。

    Python 前置知识

    • 先用 141 题模板判断是否相遇。
    • 第二阶段两个指针都每次走一步。
    • 这题的重点是入口，不只是判断有环。

    解题步骤

    1. 快慢指针找相遇点。
    2. 如果快指针走到空，说明没有环，返回 None。
    3. 令一个指针从头开始，另一个留在相遇点。
    4. 两个指针同步走，相遇处就是入口。

    完整 Python 代码

    leetcode.py
    class Solution:
    def detectCycle(self, head):
        slow = fast = head
        while fast and fast.next:
            slow = slow.next
            fast = fast.next.next
            if slow == fast:
                p = head
                while p != slow:
                    p = p.next
                    slow = slow.next
                return p
        return None

    逐行解释

    • 第一段循环找环内相遇点。
    • p = head 后开始第二阶段。
    • p 和 slow 都每次走一步。
    • 二者相遇时返回入口节点。

    示例执行过程

    3 -&gt; 2 -&gt; 0 -&gt; -4 且尾部连回 2，最终返回值为 2 的节点。

    时间复杂度

    O(n)。

    空间复杂度

    O(1)。

    易错点

    • 第二阶段不要让快指针继续走两步。
    • 没有环时要返回 None。
    • 不要返回第一次相遇点，它不一定是入口。

    ACM 模式输入输出改写

    如果输入给出 pos，可以直接输出入口位置和值；真实链表版本仍建议使用快慢指针。

    acm.py
    # input() 读取一行，split() 切分，map() 转整数，print() 输出结果
nums = list(map(int, input().split()))
pos = int(input())

if pos == -1:
    print("None")
else:
    print(nums[pos])
