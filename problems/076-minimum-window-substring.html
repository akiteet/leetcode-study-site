    Sliding Window / Hash Table

    76. Minimum Window Substring

    找出 s 中覆盖 t 所有字符的最短子串。

    Hard滑动窗口哈希表字符串

    题目白话解释

    在字符串 s 里找一段最短的连续子串，这段子串必须包含 t 中所有字符和对应次数。

    核心思路一句话

    右指针扩张直到覆盖全部需求，再移动左指针尽量缩短窗口。

    为什么会想到这个方法

    题目要求连续子串，并且要“最短”。窗口不满足时只能扩张，满足后就尝试收缩，这是滑动窗口的典型模式。

    Python 前置知识

    • Counter 可统计目标字符需求。
    • formed 记录已经满足需求的字符种类数。
    • 切片 s[l:r + 1] 可以取答案子串。

    解题步骤

    1. 统计 t 的字符需求。
    2. 右指针加入字符并更新窗口计数。
    3. 当窗口满足所有需求时，更新最短答案。
    4. 移动左指针，直到窗口不再满足。
    5. 继续扩张右指针。

    完整 Python 代码

    leetcode.py
    from collections import Counter

class Solution:
    def minWindow(self, s: str, t: str) -> str:
        need = Counter(t)
        window = {}
        required = len(need)
        formed = 0
        left = 0
        best_len = float("inf")
        best_start = 0

        for right, ch in enumerate(s):
            window[ch] = window.get(ch, 0) + 1
            if ch in need and window[ch] == need[ch]:
                formed += 1

            while formed == required:
                if right - left + 1 < best_len:
                    best_len = right - left + 1
                    best_start = left
                out = s[left]
                window[out] -= 1
                if out in need and window[out] < need[out]:
                    formed -= 1
                left += 1

        return "" if best_len == float("inf") else s[best_start:best_start + best_len]

    逐行解释

    • need：目标字符和次数。
    • formed：窗口中有多少种字符已经满足需求。
    • while formed == required：窗口有效，开始缩短。
    • best_len 和 best_start 保存最优答案。
    • 移出左侧字符后，如果低于需求，就不再满足。

    示例执行过程

    s = "ADOBECODEBANC", t = "ABC"。先扩张到能覆盖 ABC，再不断收缩，最终最短覆盖子串是 "BANC"。

    时间复杂度

    O(n)，左右指针各自最多走完整个字符串。

    空间复杂度

    O(k)，k 是字符种类数。

    易错点

    不能只判断字符是否出现，还要判断出现次数是否足够，例如 t 中有两个 A 时窗口也必须有两个 A。

    ACM 模式输入输出改写

    输入两行字符串 s 和 t，输出最短覆盖子串。
