    Sliding Window

    209. Minimum Size Subarray Sum

    找出和至少为 target 的最短连续子数组长度。

    Medium滑动窗口数组

    题目白话解释

    从数组里找一段连续区间，区间和要不小于目标值，问这样的区间最短有多短。

    核心思路一句话

    右指针扩张窗口，窗口和达标后不断移动左指针缩短窗口。

    为什么会想到这个方法

    题目是连续子数组，并且数组元素为正数。右边加入数字会让和变大，左边移除数字会让和变小，因此窗口可以稳定地向右滑动。

    Python 前置知识

    • float("inf") 可表示暂时无穷大的答案。
    • enumerate(nums) 同时拿到下标和值。
    • 窗口长度是 right - left + 1。

    解题步骤

    1. 初始化左指针和窗口和。
    2. 右指针逐步向右加入元素。
    3. 当窗口和大于等于目标值时更新答案。
    4. 移动左指针，尝试缩短窗口。
    5. 最后如果没有答案返回 0。

    完整 Python 代码

    leetcode.py
    class Solution:
    def minSubArrayLen(self, target: int, nums: list[int]) -> int:
        left = 0
        window_sum = 0
        best = float("inf")

        for right, num in enumerate(nums):
            window_sum += num

            while window_sum >= target:
                best = min(best, right - left + 1)
                window_sum -= nums[left]
                left += 1

        return 0 if best == float("inf") else best

    逐行解释

    • window_sum：当前窗口内数字和。
    • window_sum += num：右指针扩张窗口。
    • while window_sum >= target：只要达标，就尝试更短。
    • best = min(...)：保存最短长度。
    • window_sum -= nums[left]：左侧元素离开窗口。

    示例执行过程

    target = 7, nums = [2,3,1,2,4,3]。窗口扩到 [2,3,1,2] 达标后开始收缩，后面发现 [4,3] 长度为 2，是最短答案。

    时间复杂度

    O(n)，每个元素最多被右指针加入一次、被左指针移出一次。

    空间复杂度

    O(1)。

    易错点

    只有当元素都是正数时，这种“和达标就收缩”的窗口逻辑才稳定。

    ACM 模式输入输出改写

    输入第一行 n target，第二行 n 个正整数，输出最短长度。

    acm.py
    import sys

def min_sub_array_len(target, nums):
    left = 0
    window_sum = 0
    best = float("inf")
    for right, num in enumerate(nums):
        window_sum += num
        while window_sum >= target:
            best = min(best, right - left + 1)
            window_sum -= nums[left]
            left += 1
    return 0 if best == float("inf") else best

data = sys.stdin.read().strip().split()
n, target = map(int, data[:2])
nums = list(map(int, data[2:2 + n]))
print(min_sub_array_len(target, nums))
