    Dynamic Programming

    322. Coin Change

    用给定硬币凑出金额 amount，求最少硬币数。

    Medium动态规划入门完全背包

    题目白话解释

    每种硬币可以用无限次。如果无法凑出目标金额，返回 -1。

    核心思路一句话

    设 dp[x] 为凑出金额 x 的最少硬币数，枚举最后一枚硬币。

    为什么会想到这个方法

    如果最后用了面额 coin，那么前面必须凑出 x - coin，所以可以从已解决的小金额推到大金额。

    Python 前置知识

    • 这是完全背包的最少数量问题。
    • amount + 1 可作为不可能的大值。
    • 硬币可以重复使用，所以金额从小到大遍历。

    解题步骤

    1. 初始化 dp[0] = 0，其他为大值。
    2. 从 1 到 amount 遍历每个金额。
    3. 枚举每个硬币。
    4. 如果硬币不超过当前金额，就尝试更新。
    5. 最后看 dp[amount] 是否仍为大值。

    完整 Python 代码

    leetcode.py
    class Solution:
    def coinChange(self, coins: list[int], amount: int) -> int:
        dp = [amount + 1] * (amount + 1)
        dp[0] = 0

        for money in range(1, amount + 1):
            for coin in coins:
                if coin <= money:
                    dp[money] = min(dp[money], dp[money - coin] + 1)

        return dp[amount] if dp[amount] != amount + 1 else -1

    逐行解释

    • dp[money] 表示当前金额的最少硬币数。
    • dp[money - coin] + 1 表示最后加一枚 coin。
    • 不断取最小值。
    • 如果最后还是初始大值，说明凑不出。

    示例执行过程

    coins=[1,2,5]，amount=11，最少是 5+5+1，答案 3。

    时间复杂度

    O(amount * len(coins))。

    空间复杂度

    O(amount)。

    易错点

    • 无法凑出要返回 -1。
    • 硬币可以无限次使用。
    • 初始化值不能是 0，否则无法区分未计算状态。

    ACM 模式输入输出改写

    输入 n 和 amount，再输入 n 个硬币面额，输出最少硬币数。
